# 第 7-8 部分局部 PPT Storyboard v0

## Metadata

- 生产简报来源：`align/ppt_production_brief_v0.md`，状态：confirmed，确认：user/2026-05-31。
- 事实台账来源：`align/fact_ledger_v0.md`，状态：confirmed，确认：user/2026-05-31。
- 论证主线来源：`align/ppt_defense_narrative_v0.md`，状态：confirmed，确认：user/2026-05-31。
- 目标受众：课程小组成员、组长，以及后续开题报告展示听众。
- 页数策略：3 页主体页，加 1 页参考文献页供组长合并时保留或删除。
- 语言策略：PPT 主体中文；英文文献只在参考文献页出现。
- Assertion-evidence policy：relaxed。每页保留 exhibit discipline 字段；主体页允许结构图或表格承载多个来源点，但每页仍只强调一个核心判断。

## Per-Slide Storyboard

### Slide 1. 研究方法：多源互证闭环

- Action title：第 7-8 部分将研究方法和案例观察连接成一个可验证的分析闭环。
- One core point：第七章应从“列出方法名称”改为“说明每种方法对应什么材料、产出什么分析结果”。
- Source facts / claim IDs：F1、F3、F4、F16；Defense Narrative A1、A2。
- Exhibit type：process。
- Exhibit claim：文献研究、案例研究、实地走访、比较分析、归纳分析共同形成从理论到案例再到路径提炼的闭环。
- so_what_annotation：方法闭环让第八章案例分析有依据，而不是参观记录的简单整理。
- Citation need：正文页可用简短脚注标注“docx 段落 057-067；Yin 2018”，完整条目放参考文献页。
- Recommended visual form：横向闭环流程图，5 个节点依次为“文献研究 -> 案例研究 -> 实地走访 -> 比较分析 -> 归纳分析”，底部用一行标注“材料来源：原报告、实践推文、公开资料、文献”。
- Asset needs：无需外部图片；使用 PPT 可编辑形状、箭头、短标签。
- Speaking points：
  - 明确“7-8”边界：第七章研究方法，第八章初步案例分析。
  - 原文已有五类方法，但需要补上“方法-材料-产出”的对应关系。
  - 多源互证的作用是把理论文献、走访材料、公开资料和归纳维度连起来。
- Estimated time：45 秒。
- QA risk：避免把“闭环”画成过度复杂的方法论模型；不要新增 fact ledger 未确认的方法名称。
- Backup / merge / split notes：如总 PPT 页数紧张，可与 Slide 2 合并为“方法与材料来源”一页；如答辩要求方法更详细，可拆出“方法-材料-产出”三列表作为备份页。

### Slide 2. 案例基础：以深圳歌尔泰克走访材料为入口

- Action title：多源互证能避免案例分析停留在参观感想或企业介绍层面。
- One core point：本页明确采用“以深圳歌尔泰克走访材料为入口，结合歌尔股份公开资料补充背景”的口径，案例可靠性来自“走访材料 + 实践推文 + 公开资料 + 文献解释”的组合，而不是单一参观感受。
- Source facts / claim IDs：F2、F4、F5、F6、F7、F8、F11、F12、F12b；Defense Narrative A2、A5。
- Exhibit type：comparison。
- Exhibit claim：走访推文提供现场材料，公开报告补充企业事实，理论文献提供解释框架，三者互相校正。
- so_what_annotation：案例选择有事实入口，也有公开资料补强，因此可以作为技术突围研究的初步案例。
- Citation need：正文页建议标注“实践推文；歌尔 2025 年报/摘要”，完整条目放参考文献页。
- Recommended visual form：三栏证据矩阵。
  - 左栏“走访入口”：深圳歌尔泰克走访、参观、座谈、产品展示。
  - 中栏“公开资料”：业务范围、全球研发、公开报告。
  - 右栏“分析用途”：案例典型性、技术突围维度、后续路径归纳。
- Asset needs：无需截图；如后续 asset/layout 阶段允许，可考虑从实践推文或年报中提取企业资料截图，但 storyboard 阶段不生成资产。
- Speaking points：
  - 写作口径使用“以深圳歌尔泰克走访材料为入口，结合歌尔股份公开资料补充背景”。
  - 歌尔相关材料覆盖声光电、智能硬件、精密制造、全球研发布局和垂直整合。
  - 公开资料用于补强，不把所有企业集团数据直接归属于单一走访主体。
- Estimated time：55 秒。
- QA risk：主体关系容易被问到；需避免把歌尔股份整体数据直接写成深圳歌尔泰克单体数据。避免使用“全球前列”“技术封锁突破”等无强出处的绝对表述。
- Backup / merge / split notes：如组长要求减少页数，可保留三栏矩阵并删去部分事实标签；如需要更严谨，可拆出“主体口径说明”作为备份页，不进入主体 3 页。

### Slide 3. 初步观察：技术突围的五个维度

- Action title：歌尔走访材料显示技术突围依赖研发、工程、供应链、人才和全球布局的系统协同。
- One core point：第八章的案例分析应从企业介绍升级为“五维度分析框架”，为后续“路径与模式”归纳打基础。
- Source facts / claim IDs：F7、F8、F9、F10、F13、F14、F15；Defense Narrative A3、A4、A5。
- Exhibit type：diagram。
- Exhibit claim：研发设计、工程转化、产业链协同、人才组织、全球布局共同构成技术突围的可观察机制。
- so_what_annotation：技术突围不是单点技术突破，而是系统能力提升过程。
- Citation need：正文页可标注“实践推文；周文、许凌云 2023；尹西明等 2024；包玉泽等 2009”，完整条目放参考文献页。
- Recommended visual form：中心-辐射五维框架图。
  - 中心：“高科技企业技术突围”。
  - 五个外围节点：“研发设计”“工程转化”“产业链协同”“人才组织”“全球布局”。
  - 右下角加一句短注：“后续归纳路径与模式的分析基础”。
- Asset needs：无需外部图片；使用可编辑图形和图标。若后续视觉阶段使用图标，建议选择研发、制造、链路、团队、地球五类简洁线性图标。
- Speaking points：
  - 产品展示和技术积累支撑“研发设计”观察。
  - 垂直整合、精密加工和智能制造平台支撑“工程转化、产业链协同”观察。
  - 专利、青年人才、全球研发中心和客户需求导向支撑“人才组织、全球布局”观察。
  - 用新质生产力和企业创新主体理论解释这些维度的意义。
- Estimated time：65 秒。
- QA risk：不要把五个维度说成已完成证明；当前阶段只是初步案例分析和后续研究框架。避免展开企业机密、具体客户、未公开技术细节。
- Backup / merge / split notes：如 3 页主体仍显拥挤，可把 Slide 3 拆为“案例观察事实”和“五维分析框架”两页；如必须压缩到 2 页，可把 Slide 2 的案例基础并入 Slide 3 左侧小栏。

### Slide 4. 参考文献与资料来源

- Action title：参考文献为方法、理论和案例材料提供最低限度支撑。
- One core point：参考页只承担出处交代功能，主体论证仍放在前三页。
- Source facts / claim IDs：F11、F12、F13、F14、F15、F16；Production Brief 文献策略。
- Exhibit type：table。
- Exhibit claim：5 条资料分别支撑新质生产力、企业创新主体、全球价值链升级、案例研究方法和歌尔公开资料。
- so_what_annotation：参考页帮助组长合并时保留来源，同时避免主体页堆砌文献。
- Citation need：references slide only。
- Recommended visual form：两列表格，“文献/资料”和“支撑点”。中文文献按 GB/T 7714 风格；英文文献只在本页出现。
- Asset needs：无需外部资产；需要后续阶段统一参考文献格式。
- Speaking points：
  - 本页可快速带过，主要用于交代资料来源。
  - 若总 PPT 已有统一参考文献页，可把本页内容交给组长合并。
- Estimated time：15 秒。
- QA risk：参考文献格式需在后续内容保真或 deck build 前统一；歌尔 2025 年报条目需后续以实际可引用版本为准。
- Backup / merge / split notes：如总 PPT 不保留局部参考页，将本页内容移入总参考文献；如老师要求页内引用更充分，可保留本页并在前三页脚注编号。

## Ghost-Deck Check

只读行动标题形成的故事线：

1. 第 7-8 部分将研究方法和案例观察连接成一个可验证的分析闭环。
2. 多源互证能避免案例分析停留在参观感想或企业介绍层面。
3. 歌尔走访材料显示技术突围依赖研发、工程、供应链、人才和全球布局的系统协同。
4. 参考文献为方法、理论和案例材料提供最低限度支撑。

检查结果：通过。前三页能独立讲清“方法为什么改、案例为什么可靠、初步分析落在哪些维度”；第 4 页是可选参考页，不改变主体论证。

## Merge / Split Candidates

- 可合并：Slide 1 + Slide 2，可压缩为“方法与材料来源”一页，用于总 PPT 页数非常紧张时。
- 可拆分：Slide 3 可拆成“走访观察事实”和“五维分析框架”两页，用于总 PPT 允许 4 页主体时。
- 可删除：Slide 4 可删除或并入总参考文献页，但需保留 5 条资料来源给组长。

## Possible Backup-Slide Candidates

- 主体口径说明：解释“深圳市歌尔泰克科技有限公司走访材料”和“歌尔股份公开资料”的使用边界。

- 总时长建议控制在 3 分钟；如组长只给 2 分钟，优先删除 Slide 4 口播并压缩 Slide 2。
