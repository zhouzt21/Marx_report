# 第 7-8 部分修改与局部 PPT 论证主线 v0

## Defense Thesis

第 7-8 部分应证明：本课题不是泛泛谈高科技企业创新，而是通过“文献研究—案例研究—实地走访—比较归纳”的多源互证方法，把深圳歌尔走访材料与歌尔公开资料、新质生产力和全球价值链文献结合起来，说明中国高科技企业的技术突围是一种由研发设计、工程转化、产业链协同、人才组织和全球布局共同支撑的系统能力提升过程。

## Audience Contract

面向课程小组、组长和开题报告听众，这一部分需要让大家相信三件事：

1. 研究方法是可执行的：每一种方法都对应具体材料和后续产出，而不是只列方法名称。
2. 案例选择是有依据的：歌尔相关走访材料与公开资料能够支撑“高科技企业技术突围”的观察。
3. 初步分析是有边界的：不夸大企业数据，不把实践感想直接当结论，而是把它们转化为可继续研究的分析维度。

## Rubric / Committee Strategy

| 可能关注点 | 回应策略 |
|---|---|
| 是否体现《中国马克思主义与当代》的理论深度 | 在第八部分把企业创新实践连接到马克思主义生产力理论和新质生产力，而不是只写企业介绍 |
| 是否有真实实践基础 | 明确实践推文中的参观、座谈、产品展示、企业介绍和成员观察是案例材料来源 |
| 是否有文献支撑 | 参考文献覆盖新质生产力、企业创新主体、全球价值链升级、案例研究方法和歌尔年报 |
| 是否避免空泛口号 | 把“技术突围”拆成研发设计、工程转化、产业链协同、人才组织、全球布局五个可观察维度 |
| 是否存在事实风险 | 对专利、营收、领先地位等数据使用“歌尔股份及子公司公开披露”口径，不直接归属于深圳歌尔泰克单体 |

## Argument Arc

1. Situation：课题研究全球竞争下中国高科技企业的技术突围，需要既有理论框架，也有企业实践材料。
2. Gap / Problem：原第 7-8 部分已有基本框架，但方法较泛、案例分析较概括，尚未说明“方法如何支撑案例结论”。
3. Approach：以文献研究搭建理论视角，以案例研究聚焦深圳走访，以实地走访整理一手观察，以比较和归纳提炼共性路径。
4. Evidence：实践推文提供歌尔业务、全球研发布局、垂直整合、产品展示、专利和青年人才等材料；公开年报补充公司业务和经营事实；文献补充理论依据。
5. Contribution：修改后的第 7-8 部分能把“研究方法”和“初步案例分析”连接起来，为后续路径与模式归纳打基础。
6. Limitation / Answer：现阶段仍是开题和初步案例分析，不展开企业机密、财务评价或过度具体的技术结论。

## Emphasis Map

### Must Say

- “7-8”边界：第七章研究方法，第八章初步案例分析。
- 方法不是并列罗列，而是多源互证：文献、案例、走访、比较、归纳。
- 案例事实来自 PDF 实践推文和歌尔公开资料，外部文献来自 `anysearch` 检索。
- 案例分析主线是五个维度：研发设计、工程转化、产业链协同、人才组织、全球布局。
- PPT 主体中文；英文文献只在参考文献中出现。

### Can Compress

- 研究背景、研究意义、理论基础等前文内容，只保留与 7-8 相关的连接句。
- 歌尔业务范围不需要全部展开，只选声光电、智能硬件、精密制造、AR/VR/AI 眼镜等与技术突围相关的例子。
- 2026 一季报只作“已有最新公开报告”的补充，不作为正文主材料。

### Backup Only

- 详细财务数据、季度经营数据。
- 年报中各研发项目的完整列表。
- 参考文献检索过程和 anysearch 命令记录。
- “深圳市歌尔泰克科技有限公司”和“歌尔股份有限公司”之间的主体关系说明。

### Omit

- 对企业商业秘密、未公开技术细节、具体客户名称的推测。
- “市占率长期位居全球前列”等没有强出处支撑的绝对化表述。
- 与第 7-8 部分无关的整篇报告重写内容。

## Risk Map

| 风险/追问 | 预回答 | 在哪里处理 |
|---|---|---|
| 为什么案例研究法适合本课题 | 本课题关注企业技术突围机制，案例研究适合解释复杂现实情境中的“如何发生”和“为什么形成” | 第七章研究方法 |
| 走访材料会不会太主观 | 用实践推文、公开年报和文献互证，避免只依据成员感想下结论 | 第七章和第八章开头 |
| 歌尔股份数据能否直接用于深圳歌尔泰克 | 不直接等同；写作上采用“走访材料为入口，结合歌尔股份公开资料补充背景”的口径 | 第八章案例典型性 |
| 为什么技术突围不是单点技术突破 | 由产品研发、工程验证、量产交付、供应链协同、人才组织共同实现 | 第八章分析指向 |
| 参考文献为什么有英文 | 英文文献只用于案例研究方法支撑，正文和 PPT 展示仍使用中文表达 | 参考文献页或备注 |

## Action-Title Spine

这些是后续 storyboard 可用的完整句式标题，不是最终逐页安排：

1. 第 7-8 部分将研究方法和案例观察连接成一个可验证的分析闭环。
2. 多源互证能避免案例分析停留在参观感想或企业介绍层面。
3. 歌尔走访材料显示技术突围依赖研发、工程、供应链、人才和全球布局的系统协同。
4. 企业创新主体和新质生产力理论为案例分析提供了理论解释框架。
5. 这一部分的修改应为后续“路径与模式”归纳提前搭好证据链。

## Ghost Deck Test

只读行动标题也能形成完整逻辑：

- 先说明本部分要把方法和案例连接起来；
- 再说明为什么需要多源互证；
- 接着给出歌尔案例中可观察到的技术突围维度；
- 然后把案例提升到新质生产力和企业创新主体理论；
- 最后交代本部分对后续路径归纳的作用。

该逻辑可支撑 2-4 页局部 PPT，不需要做成完整开题报告。

## Storyboard Handoff Notes

- 建议 PPT 页数：3 页为宜，最多 4 页。
- 建议结构：
  1. 研究方法：多源互证框架；
  2. 案例典型性：为什么选择歌尔相关案例；
  3. 初步观察：五个技术突围维度；
  4. 可选参考文献/资料来源页，若组长要求附参考文献。
- 主体语言：中文。
- 英文文献：只放参考文献列表，不出现在主要论述标题中。
- 视觉方向：可用“方法闭环图”或“五维分析框架”表达，不需要 AI 生成图片。
